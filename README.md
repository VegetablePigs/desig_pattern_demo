## 工程简介
本项目是一个学习设计模式的demo，记录学习的源码案例

## 第一章 设计原则
* 开闭原则
  * 对扩展开放，对关闭修改
* 单一职责原则
  * 一个类只负责一个功能领域中的相对职责 
* 里式替换原则
  * 所有引用基类的地方必须能透明地使用其子类的对象 
* 依赖倒置原则
  * 依赖于抽象，不能依赖于具体的实现
* 接口隔离原则
  * 类之间的依赖关系应该建立在最小的接口上 
* 合成|聚合复用原则
  * 尽量使用合成/复合，而不是通过继承达到复用的目的 
* 迪米特法则
  * 一个软件实体应该尽可能少的与其他实体发生相互作用
